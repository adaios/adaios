## 一、项目定位

**交易课程整理 · 交易系统提炼 · 行为复盘 · 市场生态认知** — 一个 AI 知识工程项目。

将零散的付费交易课程转录文本，逐步加工为结构化的术语库、规则库和最终交易系统，最终目标是形成一套长期稳定、可执行、可复盘的**个人交易系统**。

项目后期扩展了 **12-research/** 模块，用于从**市场生态结构**的角度积累长期认知：行业结构、资本关系、IPO 动态、政策方向、资金流向等。与交易系统（解决"怎么买卖"）互补，形成市场底层结构的平行认知视角。

## 三、架构总览

整个项目分**三个层次**运转：

```
第一层：单课处理（每课独立跑完 Step 1→5  + Fusion）
              ↓
第二层：批量收敛（多课积累后融合术语、校准规则、整合系统）
              ↓
第三层：人工审核（07-manual 修正 → 按需重建下游）
```

### 数据流依赖链（单向，不可逆）

```
00-temp/ (原始下载文稿)
   │
   ▼
01-raw/ (原始归档，永久保留，只能新增不允许删除)
   │
   ▼
02-cleaned/ (最低清洗文本，保留口语/闲聊/失败案例)
   │
   ├──→ 03-glossary/*.glossary.md (逐课术语提取)
   │         │
   │         ▼
   │    Fusion ──→ 03-glossary/current/glossary.md (14 分类融合术语库)
   │         ↑
   │   07-manual/glossary/ (人工修正，融合时覆盖 AI 定义，优先级最高)
   │
   ├──→ 04-rules/*.rules.md (逐课规则提炼，保留原文金句作为备注)
   │
   └────────→ 05-system/trading-system.md (最终交易系统)
                       ↑
                 07-manual/system/ (人工系统结构，直接采用)
```

各目录之间的依赖关系：**03-glossary → 04-rules → 05-system**（上游变更破坏下游一致性，按需触发重建）

### 12-research/ 子目录说明

**核心定位：** 市场生态结构的长期认知积累。独立于课程体系，不走 Step 1→5 流程，不从 03-glossary/ 或 04-rules/ 派生。目标是建立**宏观→行业→公司→交易**的分析视角，与交易系统互补。

**核心产出类型（结构化认知表格）：**
- **产业/政策结构表** — 十五五重点方向、国家战略驱动、生命周期与交易属性映射
- **IPO 生态表** — IPO 方向→资金来源→市场作用→风险的关联分析
- **资本关系表** — 城市→产业→资本（如合肥国资、国家大基金）的对应关系
- **三/五重共振模型** — 多维度交叉判断标准（国家战略×地方国资×IPO市场×…）
- **观察池** — 长期关注的行业（★评级）和公司（含 IPO 进展跟踪）

**文件清单：**

| 文件 | 用途 | 核心内容 |
|:-----|:-----|:---------|
| `00-core-map.md` | **核心资产认知地图** | 永久框架，十五五资产结构 / IPO入口 / 城市资本 / 三重共振 |
| `01-industries.md` | **行业观察池** | ★评级行业 + 按日期的政策/资本/事件记录 |
| `02-companies.md` | **公司跟踪池** | IPO进展 / 产业地位 / 资本关系 / 关键事件 |
| `03-inbox.md` | **收件箱** | 零散想法，最灵活，定期沉淀到上层文件 |

**更新节奏：** Weekly / Monthly / Quarterly / Themes（持续维护长期主题） / Watchlist（持续跟踪行业与公司）

**与交易系统的关系：**

| | 交易系统 (05-system/) | 市场认知 (12-research/) |
|:--|:---------------------|:-----------------------|
| 解决什么问题 | 怎么买卖、何时买卖 | 市场底层结构是什么 |
| 信息源 | 交易课程转录 | 新闻、老师观点、自主分析 |
| 产出形式 | 术语/规则/操作流程 | 结构表、关系图、观察池 |
| 更新方式 | 每课 Step 1→5 | 不定期，有感而发 |
| 两者关系 | **互补** — 交易系统管执行，市场认知管选方向 |

## 五、单课处理流程

每课按**日期升序**依次处理，确认后再走下一课。

| 步骤 | 内容 | 产出文件 |
|:----:|:-----|:---------|
| Step 1 | 合并 00-temp/ 文稿 → 01-raw/ | `01-raw/YYYY-MM-DD_主题.md` |
| Step 2 | 最低限度清洗 → 02-cleaned/ | `02-cleaned/YYYY-MM-DD_主题.cleaned.md` |
| Step 3 | 提取交易术语 + 系统贡献 → 03-glossary/ | `03-glossary/YYYY-MM-DD_主题.glossary.md` |
| Fusion | 术语归入 14 分类 → current/ | `03-glossary/current/glossary.md` |
| Step 4 | 提炼交易规则 | `04-rules/YYYY-MM-DD_主题.rules.md` |
| Step 5 | 基于当前所有课程更新交易系统 | `05-system/trading-system.md` |
| 收尾 | 写入 .done 标记，清理 00-temp | `06-processed/YYYY-MM-DD_主题.done` |

### Step 3 补充说明：术语提取结构

每个 glossary 文件包含两个部分：
1. **术语定义**（名称、定义、特征、市场意义、常见误区）
2. **本课系统贡献**（按六层模块图定位：底层认知层 / 择时层 / 买入层 / 应对层 / 分析依据 / 认知与纪律）

### Fusion：14 分类体系

术语融合后归入以下 14 个分类，每个术语只出现一次：

| # | 分类 | 覆盖内容 |
|:-:|:-----|:---------|
| 1 | 交易系统总纲 | 六步闭环、执行口诀、正确vs错误交易 |
| 2 | A股市场底层逻辑 | 国家意志、政策周期、China New Story |
| 3 | 择时体系 | 活跃市值/0AMV、入场分级、2.3%离场法则 |
| 4 | 选股与配置 | 曼城阵容、六大赛道、填坑出坑 |
| 5 | N型结构与节奏 | 三层节奏体系、吸呼感知法 |
| 6 | 量价与筹码 | 缩量真实、3/4阴量线、筹码纯度 |
| 7 | B1买点体系 | B1/B2/B3/SB1、买入三问 |
| 8 | 入场方式与战法 | 暴力特噗、深水炸弹、补票战法 |
| 9 | 止损体系 | 只输一根K线、止损三级别 |
| 10 | 止盈与收队 | BBI线减仓一半、见顶信号 |
| 11 | 仓位管理 | 四不原则、两枪原则 |
| 12 | 市场参与者认知 | 机构vs游资vs量化、跟庄三条件 |
| 13 | 交易心理与纪律 | 核心认知、常见错误 |
| 14 | 特色词汇与口诀 | 黑话表、所有口诀 |

## 六、核心工作原则

1. **绝不删除 01-raw/** 原始数据 — 永久保留
2. 优先提炼**明确规则**，优先**风险控制**
3. 不做荐股，不主观发挥，不过度扩展系统复杂度
4. 保持系统简单、明确、可执行
5. **上游变更按需触发重建**（由用户确认后执行）
6. 保留原文**口语、闲聊、失败案例**（用户日常复盘的重要素材）
7. 规则提炼**保留原文金句作为备注**
8. 07-manual/ 有修正时，AI **只确认收到，不直接修改对应产出文件**

## 八、给其他 LLM 的指引

如果被要求参与此项目：

1. **先读 `CLAUDE.md`** — 它包含所有操作指令、目录权限规则、文本处理原则
2. **先读 `PROJECT_SUMMARY.md`**（本文件）— 了解项目全貌
3. **遵循数据流方向** — 永远从上游向下游处理，不要反向操作
4. **尊重 07-manual/ 不可变性** — 那是用户的领地

6. **保留口语和案例的温度** — 不要过度压缩用户的经验表达
