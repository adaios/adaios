### 12. 融合流程缺陷：追加模式缺少归一化检查

**发现日期：** 2026-07-11
**来源：** BBI→双线术语冲突追溯

**根因：**
当前 Phase A（融合术语重建）是**追加模式**——新课程有术语就追加进对应分类，没有就跳过。
它漏了第三步：**全文档扫描旧表述**。

```
融合流程（当前）
  遍历新课程 glossary → 新术语？→ 追加到对应分类
                       → 无新术语？→ 跳过
                       
融合流程（应有）
  遍历新课程 glossary → 新术语？→ 追加到对应分类
                       → 无新术语？→ 跳过
                       → 无论有无，检查新课程是否引入/升级了某个指标
                          → 如是，全文档扫描替换旧表述（如 BBI→双线）
```

**结果：** 双线战法作为新分类 #61 行进来了，但老分类 #10 "BBI线上减仓一半"、止损规则"BBI拐头止损"、执行口诀"看BBI线" 等十余处未同步更新。

**修正建议：** 在 CLAUDE.md 融合流程（Phase A）中增加"术语归一化检查"步骤。

**严重程度：** 流程级。当前 BBI 遗留不影响交易执行（BBI 和白线用法互通），但长期累积会导致术语库越来越分裂，后续新课程再引入指标升级时问题放大。
