# ADAI 设计原则

> 版本：v1 — 2026-07-10
> 产品定位：Personal Context Platform
> 一句话：记录今天。理解过去。帮助未来。

---

## 核心哲学

### 不是聊天机器人

ADAI 不是 ChatGPT，不是微信，不是任何需要"对话"的东西。它是**一个人打开手机后第一个打开的 App**，而不是"有问题时才打开的工具"。

### 不要用户分类

用户永远不需要选择"这是交易"还是"这是日记"。记录就是记录。后台 AI 自动理解上下文、自动分类、自动关联。就像你跟朋友说话不用先声明"我现在要说一个交易相关的事情"。

### 一个页面

没有 Tab。没有 BottomNavigation。没有多级页面。
产品只有一个页面：**Today**。

---

## 页面结构

### 唯一页面的三个区域

```
┌──────────────────────────────┐
│ 7月10日                 ⊙    │  ← Top Bar: 日期 + Timeline 入口
├──────────────────────────────┤
│                              │
│ 07:00  ·  分析               │
│ 早上好。今天下午18:00...      │
│                              │
│ 07:00  ·  分析               │
│ 京东方A 回踩支撑位...         │
│                              │
│ 09:20  ·  交易               │
│ 今天买了立昂微。              │  ← Today Feed (统一卡片流)
│                              │
│ 14:00  ·  生活               │
│ 下午喝凤凰单丛。              │
│                              │
│ 14:30  资 讯                 │
│ Figure AI 发布新原型机...    │
│                              │
│ 21:01  ·  分析               │
│ 今天整体符合预期...           │
│                              │
├──────────────────────────────┤
│ [🎤]  ┌─────────────────┐  ⊕ │  ← Input Bar (语音/文字)
│       │ 记一笔…           │    │
│       └─────────────────┘    │
└──────────────────────────────┘
```

**① Feed 上方**：日期（左）+ Timeline 图标（右）。没有标题、没有 logo。

**② Today Feed**：从早晨简报开始，到一天中的所有记录。所有条目都是**统一的卡片样式**，通过左边色条和标签区分角色。无分割线、无分区标题。

**③ Input Bar**：一行三列 — 语音/文字切换（左）+ 输入区域（中）+ ⊕/发送（右）。

## 卡片设计

### 统一容器

所有 Feed 条目共享同一个卡片容器（16px border-radius、#1A1A1A 底色），通过以下元素区分角色：

| 角色 | 左侧标识 | 标签 | 文字风格 |
|------|---------|------|---------|
| **你的记录** | 无 | 无 | 16px, w500 |
| **AI 分析** | 绿色 5px 竖条 + 绿色小圆点 | "分析" (绿, 9px, letter-spacing) | 15px, w400 |
| **资讯** | 蓝色 5px 竖条 + 蓝色小圆点 | "资讯" (蓝, 9px, letter-spacing) | 15px, w400 |

### 卡片布局

```
┌────────────────────────────────┐
│ 09:20  ·  交易  ·  持仓        │  ← 时间 + 标签 (同一行, 左上角)
│                                │
│ 今天买了立昂微。                │  ← 内容主体
│                                │
│ [ 最近交易 ] [ 持仓分析 ]       │  ← 来源标签 (仅 AI/资讯下)
└────────────────────────────────┘
```

- **时间列取消** — 时间 + 标签合并到内容上方一行
- **左色条用于 AI/资讯** — 5px，一眼可辨
- **用户记录无装饰** — 视觉最轻，只有文字
- **卡片间距 6px** — 相邻卡片 12px 呼吸感

## 输入栏设计

### 语音 / 文字 双入口

```
[🎤]  ┌─────────────────────┐  [⊕]
       │ 记一笔…              │
       └─────────────────────┘
       ↓ 点击左按钮切换
[⌨]  ┌─────────────────────┐  [⊕]
      │   按住 说话           │
      └─────────────────────┘
      ↓ 长按后
[⌨]  ┌─────────────────────┐  [⊕]
      │ 🎤 松开 发送          │  (绿色反馈)
      └─────────────────────┘
```

- 左按钮：麦克风（语音模式）↔ 键盘（文字模式），点按切换
- 文字模式：聚焦后键盘弹出，支持多行，Enter 发送
- 语音模式：长按"按住 说话"，松开后发送
- 右按钮：始终 40×40，无位移。无文字时 ⊕（打开附件菜单），有文字时 ↑（发送）
- 高度固定 40px（整栏 56px 含 padding）
- 附件菜单（⊕ 点击弹出）：图片 / 语音 / 文件 / 链接

---

## Timeline (时间线)

不是一级页面。是右上角图标 → BottomSheet 弹窗（75% 高度）。

左侧时间轴竖线 + 圆点，日期分组。

## 视觉语言

### 深色模式优先

| Token | 色值 | 用途 |
|-------|------|------|
| darkBg | `#0E0E0E` | 背景 |
| darkSurface | `#1A1A1A` | 卡片 |
| darkSurface2 | `#232326` | 输入栏、按钮 |
| darkBorder | `#2C2C2E` | 分割线 |
| darkGrey1 | `#F0EDE9` | 最高权重文字 |
| darkGrey2 | `#D4D0CB` | 高强调 |
| darkGrey3 | `#B5B0AA` | 正文 |
| darkGrey4 | `#908B85` | 次要文字 |
| darkGrey5 | `#66615C` | 三级文字 |
| darkGrey6 | `#45423E` | 占位符 |

暖灰色系，中性偏暖。6 级灰色梯度的目的是给不同层级的内容精确的视觉权重。

### 无科技蓝

禁止使用蓝色渐变、蓝色主色调、发光效果。蓝色仅用作资讯来源的标识色（muted blue `#5299FF`）。

### 排版

- 34px "今天"（节日场景，已取消）
- 22px 时间线弹窗标题
- 15-16px 正文
- 11-12px 时间、标签
- 9px 分析/资讯标签

无衬线字体。正文 `FontWeight.w400`。用户自己的记录 `FontWeight.w500`。

### 禁止清单

- 没有 Tab / BottomNavigation
- 没有聊天气泡
- 没有左右对话分栏
- 没有拟物
- 没有花哨动画
- 没有科技蓝 / 渐变
- 没有 "Brain" / "AI" / "Knowledge" / "Capture" 等产品词
- 没有 Dashboard KPI / 统计数字
- 没有分类选择器

## 交互原则

### 顺滑，但不花哨

- 每次打开有 600ms 的 fadeIn + slideUp 入场过渡
- 新条目出现时带 300ms 的 easeOut 滚动到最新
- 卡片悬停有轻微上浮（桌面端）
- 输入模式切换有 200ms AnimatedSwitcher
- 没有弹簧动画、没有旋转、没有粒子

### 输入即记录

用户在输入框中打字时不需要选择"语音还是文字"以外的任何东西。写的内容进入 Feed 后自动成为当天的一部分。

### 不需要保存按钮

无"保存""提交""确认"等事务性操作。输入就是记录，Enter 就是保存。

---

## 数据原则

- **无后端。无接口。无数据库。**
- 所有 Mock 数据在 `lib/data/mock_data.dart` 中
- 不使用任何状态管理（Provider / Riverpod / Bloc）
- `setState` 足以管理当前页面的数据
- 数据模型最小化：`FeedItem`、`TimelineEntry` 两个模型

---

## 情绪板

产品给人的感觉应该是：

> 打开 → 看到今天的日期 → 从上往下读今天发生了什么 → 想写点什么就写下 → 关闭。

不是：
> 打开 → 看到"上午好，阿呆" → 看到 Dashboard KPI → 不知道应该点哪里 → 关闭。
