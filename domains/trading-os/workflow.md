# Trading OS — 工作流

## 策略交易主流程

```
AI/用户 → 策略信号 → 风控校验 → 订单生成 → 订单路由 → 成交回报 → 数据归档
    ↑                                                                     |
    └────────────────────── 策略持续评估 ← 行情更新 ←─────────────────────┘
```

1. **策略触发**：行情变化或定时事件触发策略评估
2. **信号生成**：策略计算买卖信号与仓位建议
3. **风控校验**：校验信号是否触及风控规则（仓位/频率/金额限制）
4. **订单生成**：通过校验的信号转为订单指令
5. **订单路由**：订单发往对应账户的交易所
6. **成交回报**：接收交易所成交结果，更新持仓与资金
7. **数据归档**：交易记录写入 adai-data

## 回测流程

```
策略定义 → 历史行情加载 → 逐笔模拟 → 信号评估 → 模拟下单 → 绩效统计
                           ↑                              |
                           └─────────── 时间推进 ─────────┘
```

1. 加载策略参数与历史数据
2. 按时间序列逐笔推进，模拟市场状态变化
3. 策略在每一时间点评估入场/出场条件
4. 生成模拟订单，考虑滑点和手续费
5. 回测结束时计算绩效指标

## 行情处理流程

```
行情源 → 协议解析 → 归一化 → 实时推送 → 策略引擎
   ↕
数据持久化 → adai-data
```

- 多数据源并行接入（券商 API、WebSocket、REST 轮询）
- 统一内部行情格式，屏蔽不同源的差异
- 实时行情推送至策略引擎，历史行情供回测使用

## 异常处理场景

| 场景 | 处理方式 |
|------|----------|
| 行情断连 | 启用缓存行情，尝试重连，超时后暂停策略 |
| 下单失败 | 记录失败原因，通知用户，不自动重试 |
| 风控触发 | 冻结对应策略，记录审计日志 |
| 账户断连 | 暂停所有关联策略，等待重连认证 |

## 与 adai-core 的协作流程

```
adai-core (application 层)          trading-os
      │                                │
      │ 1. context.load("当前持仓")    │
      │──────────────────────────────→ │
      │ 2. 返回持仓快照                │
      │ ←─────────────────────────────│
      │ 3. strategy.evaluate(ctx)      │
      │──────────────────────────────→ │
      │ 4. 返回策略信号                │
      │ ←─────────────────────────────│
      │ 5. 确认/拒绝信号（用户/AI）     │
      │ 6. order.execute(signal)       │
      │──────────────────────────────→ │
      │ 7. 返回订单状态                │
      │ ←─────────────────────────────│
      │ 8. memory.save(trade_result)   │
```
