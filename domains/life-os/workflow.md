# Life OS — 工作流

## 日程管理流程

```
AI/用户 → 日程创建意图 → 冲突检测 → 时间确认 → 保存 → 同步第三方
                  ↓
            重复规则展开
```

1. 用户或 AI 表达创建日程意图（"下周一三点开会"）
2. 检测与已有日程是否冲突
3. 如有冲突，提供替代方案
4. 用户确认后保存，按规则展开重复日程
5. 同步至已关联的第三方日历（如 Google Calendar）

## 习惯追踪流程

```
每日提醒 → 用户记录 → streak 更新 → 里程碑检测 → 激励反馈
                                              ↓
                                        习惯洞察
```

1. 定时提醒用户记录当日习惯
2. 用户确认或自动记录（设备数据）
3. 更新连续完成天数（streak）
4. 达到里程碑时生成激励反馈
5. 定期生成习惯完成趋势报告

## 健康数据处理流程

```
设备数据 → API 接入 → 归一化 → 基线对比 → 异常检测 → 存储
                    ↓                               ↓
              数据清洗                           adai-data
```

1. 从智能设备/健康应用接入原始数据
2. 统一数据格式
3. 与用户基线对比，标记异常
4. 正常数据和异常数据分别存储处理
5. 异常触发通知（"心率持续偏高"）

## 与 adai-core 的协作流程

```
adai-core (application 层)            life-os
      │                                │
      │ 1. "今天有什么安排？"           │
      │──────────────────────────────→ │
      │ 2. schedule.today()            │
      │ ←─────────────────────────────│
      │    → context.set(schedule)      │
      │                                │
      │ 3. "这周健康数据怎么样？"       │
      │──────────────────────────────→ │
      │ 4. health.weeklyReport()       │
      │ ←─────────────────────────────│
      │    → context.set(report)        │
      │                                │
      │ 5. 定时触发器 → remind()        │
      │ ←─────────────────────────────│
      │ 6. 通知用户                      │
```
