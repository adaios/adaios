# Project OS — 核心概念

## 核心实体

### Idea（想法）
未加工的灵感或提议。包含标题、描述、来源、标签、优先级、状态（待评估/已采纳/已搁置/已拒绝）。想法可以升级为项目。

### Project（项目）
有明确目标的执行单元。包含名称、目标、时间范围、状态（规划中/进行中/暂停/已完成/已取消）、优先级、关联标签。

### Task（任务）
项目内的最小可执行单元。包含标题、描述、状态（待办/进行中/已完成/已阻塞）、预估工时、实际工时、截止日期、负责人、依赖任务、优先级。

### Milestone（里程碑）
项目中的关键节点。包含名称、日期、交付物描述。里程碑达成后触发复盘。

### Sprint（迭代周期）
短期执行周期。包含起止日期、目标、关联任务列表。用于分阶段推进项目。

### Retrospective（复盘）
项目或迭代结束后的回顾。包含做得好/做得不好/改进行动。

## 领域概念

### 项目状态机
```
想法 → 项目规划 → 执行（迭代循环）→ 完成 → 复盘
                        ↑               ↓
                    未完成目标←──── 调整重规划
```

### 任务依赖类型
- **前置依赖**：A 必须在 B 之前完成
- **阻塞依赖**：A 依赖 B 的输出才能继续
- **资源依赖**：需要特定资源（信息、设备、协作）

### 进度追踪
- **完成百分比**：基于已完成任务/总任务
- **燃尽图**：剩余工作量 vs 时间
- **迭代速率**：每个迭代完成的任务点数
- **阻塞率**：被阻塞任务数/总任务数

### AI 辅助场景
- 想法整理：AI 辅助分类、关联已有项目
- 任务分解：AI 自动将目标分解为任务列表
- 进度预警：AI 识别进度落后风险
- 模式发现：AI 总结项目历史中的有效工作模式
