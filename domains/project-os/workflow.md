# Project OS — 工作流

## 想法到项目流程

```
想法捕捉 → AI 分类 → 关联已有项目 → 评估 → 立项 → 规划
                              ↓
                        无关联 → 新建项目
```

1. 用户随时记录想法（"做一个个人博客"）
2. AI 自动分类并关联已有项目
3. 用户评估是否采纳
4. 采纳后创建项目，进入规划阶段

## 项目执行流程

```
规划 → 创建任务 → 迭代执行 → 每日检查 → 完成 → 复盘
   ↑                                    ↓
   └────────── 范围调整 ←──────────────┘
```

1. 规划阶段：定义目标、里程碑、时间线
2. AI 辅助将目标分解为任务
3. 每个迭代执行一批任务
4. 定期检查进展，识别阻塞
5. 完成后复盘，形成经验写入 Memory

## 与 adai-core 的协作流程

```
adai-core (Context Engine)              project-os
      │                                      │
      │ 1. Context.load(project_id)          │
      │    → Identity(工作方式偏好)          │
      │    → Timeline(近期时间投入)          │
      │    → Memory(类似项目经验)            │
      │────────────────────────────────────→ │
      │ 2. 返回 Project Context Package      │
      │ ←────────────────────────────────────│
      │                                      │
      │ 3. "这个项目进展如何？"              │
      │────────────────────────────────────→ │
      │ 4. project.summary(id)               │
      │ ←────────────────────────────────────│
      │    → Context.update(response)         │
```
