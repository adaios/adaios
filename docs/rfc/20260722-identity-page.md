# RFC: 身份页（Identity Page）

> 日期：2026-07-22
> 状态：草案
> 涉及：前端（Flutter）+ 后端（Spring Boot）

---

## 1. 目标

新增身份页，让用户在 App 内查看和编辑个人档案（Identity Profile）。

当前身份数据存在 `data/identity/profile.md`，后端 `ContextEngine` 在聊天时读取并注入 system prompt，但**用户在 App 内无法看到它**身份是什么，也无法修改。

身份页解决：**"AI 怎么看我的？我想改，在哪改？"**

---

## 2. 页面布局

### 2.1 页面位置

底部导航栏新增第 4 个 Tab（👤 身份），与 🏠 Feed / 💹 交易 / 🧠 记忆 并列。

```
┌─────────────────────────────┐
│  🏠 Feed  💹 交易  🧠 记忆  👤 身份  │
└─────────────────────────────┘
```

**设计要点：**
- 所有信息**只读展示**，不可直接编辑
- 标签显示为 `Chip` 样式
- 规则显示为不可交互的 Switch（开状态显示绿色，关状态灰色）
- 底部固定按钮 "编辑个人档案"

### 2.3 编辑模式

点击「编辑」后，页面原地切换为编辑状态（不跳新页面）：

```
┌──────────────────────────────┐
│  7/22·周二                   │
│                              │
│  编辑个人档案                │
│                              │
│  称呼 *                      │
│  ┌────────────────────────┐  │
│  │ 阿呆                   │  │
│  └────────────────────────┘  │
│                              │
│  沟通风格                     │
│  ┌────────────────────────┐  │
│  │ 简洁、直接              │  │
│  └────────────────────────┘  │
│                              │
│  专注领域                     │
│  ┌────────────────────────┐  │
│  │ 半导体、国产替代、       │  │
│  │ 成长股投资               │  │
│  └────────────────────────┘  │
│                              │
│  AI 协作规则                  │
│  ┌─ 交易类操作需确认 ────[●]─┐ │
│  ┌─ 日常记录自动处理 ────[●]─┐ │
│                              │
│  关注标签                     │
│  │ 投资 [×]  半导体 [×]     │ │
│  │ 科技 [×]  个人成长 [×]   │ │
│  │ [+ 添加]                 │ │
│                              │
│  [ 💾  保存 ]  [ 取消 ]       │
└──────────────────────────────┘
```

**交互要点：**
- name 为必填（显示 * 号），为空时保存按钮置灰
- 标签支持删除（点 ×）和添加（点 "+ 添加" 弹出输入框）
- 规则使用 `SwitchListTile`，点击切换
- 保存 → `PUT /api/v1/identity` → 成功后切回展示模式
- 取消 → 放弃编辑，切回展示模式

## 3. 后端 API

| 方法 | 路径 | 请求体 | 响应 | 说明 |
|:----|:-----|:-------|:-----|:-----|
| `GET` | `/api/v1/identity` | — | `IdentityProfile` | 读取 |
| `PUT` | `/api/v1/identity` | `IdentityProfile` | `IdentityProfile` | 全量覆盖 |

详见 `api-spec.md §8`。

### 后端改动

| 文件 | 改动 | 说明 |
|:-----|:-----|:------|
| `IdentityRepository.java` | 新增 `save(IdentityProfile)` | 端口定义 |
| `IdentityFileRepository.java` | 实现 `save()` | YAML frontmatter 序列化回写 |
| `IdentityController.java` | **新建** | GET + PUT 两个端点 |

## 5. 不做的事（Future Scope）

| 功能 | 原因 |
|:-----|:------|
| 头像上传 | 需要后端支持图片存储 + 前端 picker，非 MVP |
| 多身份切换 | 当前只支持一个 profile |
| 历史版本回滚 | profile.md 有 git 管理，不需要 UI 层面实现 |

## 6. 验收条件

- [ ] 底部导航有 4 个 Tab，身份页可点击进入
- [ ] 展示模式正确渲染从后端读取的身份数据
- [ ] 编辑模式可编辑称呼、风格、规则、标签
- [ ] 保存后刷新展示模式，数据一致
- [ ] 网络错误时显示错误态 + 重试按钮
- [ ] Dark/Light 主题一致
