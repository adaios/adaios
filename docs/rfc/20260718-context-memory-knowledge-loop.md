# RFC: Context 闭环 — 记忆回读 + 知识召回

> 日期：2026-07-18
> 状态：草案
> 领域：Kernel / Context Engine

---

## 问题

当前 Context Engine 组装 Prompt 时只用了三样东西：

1. Identity（你是谁）
2. 今日会话历史（最近 5 条记录）
3. Domain Contributor（当前只有持仓数据）

但有两样核心资产被忽略：

- `data/memory/` — 你过去一个月沉淀的 AI 理解摘要，从未被回读到新会话
- `os/trading-os/04-rules/` — 87 篇你整理的个人交易规则，AI 回答交易问题时不知道这些知识

**后果：** AI 每次回答都像"第一次认识你"——不知道你长期关注什么，不知道自己有你的知识库可用。

---

## 方案：Context Engine 双向增强

在现有 ContextContributor 插件机制上，新增两个贡献者，不破坏现有架构：

### 新增 A：MemoryContextContributor

**职责：** 将过去 N 天的记忆摘要注入上下文。

**触发条件：** 始终生效（default contributor）。

**注入内容：**

```
## 近期记忆摘要（最近 7 天）

- [7月12日] 因半导体国产替代买入立昂微，担心追高
- [7月12日] 立昂微上涨3%，验证逻辑，继续持有
- [7月18日] 问过天气、做过调仓（卖出永兴材料→买入TCL科技）
```

**效果：** AI 知道"你最近在关注什么、操作了什么"，问答有连续性。

**成本：** 1 个新类 + 1 次文件读取。

---

### 新增 B：TradingKnowledgeContributor

**职责：** 当识别到交易相关问题时，去 `os/trading-os/04-rules/` 搜索匹配的知识文件，注入相关内容。

**触发条件：** `scene="trading"` 或用户输入包含交易关键词（买入/卖出/止损/B1/KDJ/等）。

**搜索逻辑：** 关键词匹配规则文件名 + 规则内容中的「描述」字段。

**注入内容（精简，非全文）：**

```
## 你的交易知识库（匹配结果）

找到 3 篇相关规则：

### B1（买点1）入场三依据
买入前必须同时确认三个依据：
N型上涨结构 + 量能配合（放量吸筹→缩量回调）+ KDJ大幅值（<13~20）
三者缺一不可。

### 只输一根K线
买入后以进场K线最低价下方1%设止损，收盘跌破就走。
这是永不套牢法的基石。

### 别追高——跌了才是机会
不在股价拉升后追入，坚持在回调到位的 B1 位置入场。
```

**效果：** AI 回答交易问题时，引用你**自己整理**的交易规则，而不是通用大模型泛泛而谈。

**代价：** 1 个新类 + 文件系统搜索 + 关键词匹配。

---

### 数据流变化

```
当前：
  Record → Context Engine → [Identity + 会话历史 + 持仓] → Prompt → AI 回答

增强后：
  Record → Context Engine → [Identity + 会话历史 + 持仓
                              + 记忆回读(过去7天)            ← 方向二
                              + 知识召回(os/trading-os/)]     ← 方向一
                            → Prompt → AI 回答
```

---

### 不会改变的东西

- **RecordController / API 契约** — 完全不变，前端无感
- **Card UI / 状态机** — 不动
- **IntentRecognizer** — 不动
- **MemoryService / FileStorage** — 复用现有能力
- **测试** — 现有 100 个后端测试不受影响

### 关键问题

**Q: 知识文件有 87 篇，全塞进 Prompt 会超 Token 吗？**
A: 不会。只搜索关键词匹配的 1-3 篇，每篇只提取「类型 + 描述」字段，不塞全文。预计注入 500-1000 字符，远低于 Context 限制。

**Q: 搜索精度够吗？**
A: 先用关键词匹配（文件名+内容标题字段），后续可升级为向量搜索。

**Q: os/ 目录独立性是否被破坏？**
A: 不破坏。adai-core 只读取 `04-rules/` 下的 `.md` 文件，不写入。os/trading-os/ 的 Git 生命周期完全不受影响。

**Q: Memory 回读会不会让 Prompt 太长？**
A: 7 天记忆平均 10-20 条摘要，每条一句话，约 300-500 字符。可控。
